# Claude Design 프롬프트 — Detail Card 4탭 리디자인

**사용법**: 전체 블록을 Claude Design 새 대화에 복사. `index.html` 첨부 + `DESIGN_SYSTEM.md` 첨부.

---

저는 고연승T 수능 영어 학원의 매니저 hyun 이에요. 학생 일차 성적표의 **Detail Card** 를 손봐주세요. 이 카드는 학생뿐 아니라 **학부모도 함께 보는** 상세 리포트입니다.

## 범위
공유 카드 4종은 이번 범위 밖. **Detail Card 1장 + 내부 4탭만** 봅니다.

탭 구성:
1. 이번 주 (THIS WAVE) — 요약·히어로 오리·코멘트
2. 업적 (BADGES) — 5개 배지 + Club stamp
3. 분석 (ANALYSIS) — 유형별 정답률·추이 차트
4. 꾸미기 (DRESS UP) — 5슬롯 커스터마이징 (탭 프레임만. 내용은 다른 프롬프트에서)

## 준수할 기준
첨부 `DESIGN_SYSTEM.md` 의 팔레트·타이포·오리 본체 좌표·카피 톤. 특히:
- 탭 스위처는 CSS-only radio 패턴 유지 (JS 없음)
- 기존 DOM 구조·클래스명·id 유지 (탭 내용 리스타일링만)
- 영어 라벨 + 한국어 본문 조합
- Detail 히어로 viewBox 4:3 비율 유지

## 지금보다 나아지길 바라는 것
1. **탭 전환이 "또 눌러보고 싶다"로 느껴지게** — 지금은 정보 전환만
2. **학부모 친화성** — 한눈에 "우리 애가 이만큼 했구나" 가 오게
3. **분석 탭의 '숫자'가 위협적이지 않게** — 성적표 같은 공포감 제거, 성장 그래프 느낌
4. **업적 탭이 '컬렉션' 느낌** — 지금은 리스트, 앞으로는 전리품 진열장
5. **이번 주 탭에서 학생 코멘트가 살아있게** — 교사 코멘트가 형식적이지 않게

## 꾸미기 탭은 왜 스킵?
꾸미기 탭 내부 디자인(70종 아이템 그리드·슬롯 패널·프리뷰)은 `dressup-tab.md` 에서 별도로 다룹니다. 이 프롬프트에서는 **꾸미기 탭이 다른 3탭과 어울리는지** 프레임 수준만 확인해주세요.

## 레퍼런스 (참고)
- Apple Fitness 주간 요약 → 학부모가 싫어하지 않는 "지표 + 격려"
- Strava Stats → 차트가 압박보다 성장 스토리로 읽히게
- Pokemon Go 주간 리포트 → 내가 얼마나 성장했나
- 현대차 블루링크 월간 리포트 → 점잖은 차트

## 디자인 원칙
- 과한 네온·파스텔 금지 (서퍼 차분 무드)
- 차트는 gold/blue 2색 조합, 축 label 은 paper-dim
- 배지는 티어별 시각 공식을 차트/코멘트에도 일관적으로 반영
- 한국어 존댓말 기본, 학부모가 봐서 민망하지 않은 카피

## 산출물
- Detail Card 한 장의 4탭 각각 **리뉴얼 안 A/B**
- 탭 간 "이동 감성" (트랜지션은 최소, 정보 계층이 자연스럽게 이어지게)
- 각 탭 헤드라인 카피 제안 2안
- 학부모 시점 "이 탭을 보고 놓치지 말아야 할 정보 1줄" 각 탭 별로

## 제약
- 외부 차트 라이브러리 금지. 인라인 SVG 로만 차트 구성
- 이 mockup 은 cosmetic only, 학업 성능 영향 없음

먼저 탭별 **현재 상태 간단 평가**(각 3줄) + **학부모가 덜 관심 가질 만한 탭 1개 + 왜** + 다음 단계 제안을 주세요. 이어서 A/B.
